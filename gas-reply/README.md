# eBay バイヤー返信システム — GAS デモ（F1〜F3）

> 5/21 デモ用。**メッセージ取得 → AI返信案生成 → Discord通知** までを動かす。
> ワンクリック送信（F4）は次フェーズ（Discord Bot 必要）。

## このデモが解決する3つの課題

| 課題 | 対応機能 | デモで見せられること |
|---|---|---|
| ① 英語の翻訳・読解を自動化 | F2（Claude API） | 英語メッセージを理解し日本語/英語で返信案を生成 |
| ② 顧客への返信をある程度自動化 | F2 | バイヤーのメッセージに対しAIが返信文を下書き |
| ③ ワンクリックで作業を楽に | F3（通知）→ F4（送信ボタン） | デモは通知まで。ボタン送信は次フェーズ |

## デモ実行手順（最短）

### 1. GAS プロジェクト作成
新規スプレッドシート → 拡張機能 → Apps Script。
このフォルダの `.gs` を全部貼り付け（または clasp push）。

### 2. スクリプトプロパティ設定（デモ最小構成）
GAS エディタ → 歯車 → スクリプトプロパティ:

| プロパティ | デモでの値 |
|---|---|
| `ANTHROPIC_API_KEY` | Claude API キー（しゅんすけ提供）。**これがあるとAI返信が実際に生成される** |
| `DISCORD_WEBHOOK_URL` | テスト用 Webhook（無くてもlogsに出るので動作確認は可能） |

> eBay 系プロパティ（`EBAY_*`）はモックデモには不要。本番 F1 で必要。

### 3. シート初期化
スプレッドシートを開き直す → メニュー「💬 eBay返信BOT」→「① シート初期化」。
`返信ログ` `logs` `_seen_messages` `在庫発送_作業` が作られる。

### 4. デモ実行
メニュー →「▶ デモ: モックメッセージで実行」。

期待される動作:
- モック3件（英語2件・日本語1件）が処理される
- 各メッセージに対しAI返信案が生成される（`ANTHROPIC_API_KEY` 設定時）
- Discord に「📩 新着バイヤーメッセージ + 🤖 AI返信案」が投稿される（Webhook設定時）
- `MOCK-ORDER-002` 紐付きは `在庫発送_作業` シートへ、他2件は `返信ログ` へ記録
- 再実行しても重複しない（`_seen_messages` で排除）

### 5. リセット（再デモ用）
GAS エディタで関数 `resetMockSeen` を実行 → 処理済台帳をクリア → 再度デモ可能。

## 本番への接続（デモ後）

1. eBay Trading API の Auth'n'Auth トークン等 `EBAY_*` プロパティを設定
2. `pollMessages()` を実行 → 実メッセージで F1 動作確認
3. F1 の `parseMessages_` を実レスポンスに合わせ微調整（XML階層確認）
4. F4（ワンクリック送信）実装には Discord Bot + Interactions Endpoint が必要
   → 詳細は `../REQUIREMENTS_buyer_reply.md` の §5 アーキテクチャ参照

## 制約・注意

- GAS 6分実行制限。1回のポーリングで大量メッセージが来る場合はバッチ分割済（F1は10件ずつ取得）
- Claude API は従量課金。ポーリングごとに新着分だけ呼ぶ（重複排除で無駄打ち防止）
- AI返信は**必ず人間が確認してから送信**（自動送信はしない設計）
